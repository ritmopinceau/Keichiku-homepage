# 〇〇建設・不動産 コーポレートサイト

建築工事(個人向け・法人向け)と不動産(売買・賃貸・買取・管理)をワンストップで扱う会社のコーポレートサイトです。
Next.js (App Router) + TypeScript + Tailwind CSS で構築しています。

## 1. ローカル環境での起動方法

Node.js (18.18以降推奨) がインストールされたPCで、プロジェクトフォルダに移動して以下を実行してください。

```bash
cd kenchiku-fudousan-hp

# 依存パッケージのインストール（初回のみ）
npm install

# 開発サーバーの起動
npm run dev
```

起動後、ブラウザで `http://localhost:3000` を開くと確認できます。

その他のコマンド:

```bash
npm run build   # 本番用ビルド（型エラー・ビルドエラーの確認にも使えます）
npm run start   # ビルド後の本番モードでの起動
npm run lint    # ESLintによるコードチェック
```

## 2. ディレクトリ構成（概要）

```
app/         各ページ（App Router）
components/  共通コンポーネント（layout / sections / cards / templates）
data/        会社情報・サービス内容・施工事例・物件情報などのデータ
lib/         SEO用メタデータ生成、お問い合わせ送信処理などの共通ロジック
public/      画像などの静的ファイル
```

## 3. よく変更する項目と変更場所

### 会社名・住所・電話番号・営業時間・許認可番号など
[`data/siteConfig.ts`](data/siteConfig.ts) の `siteConfig.company` を編集してください。
電話番号はここを1箇所変更するだけで、ヘッダー・フッター・スマホ下部固定メニュー・お問い合わせページなど
サイト全体に反映されます。

```ts
company: {
  name: "株式会社〇〇",          // ← 確定した会社名に変更
  phoneDisplay: "00-0000-0000", // ← 表示用の電話番号
  phoneHref: "tel:0000000000",  // ← tel:リンク用（ハイフンなし）
  ...
}
```

### サイト名・SEOキーワード・地域名（〇〇市・〇〇県）
[`data/siteConfig.ts`](data/siteConfig.ts) の `siteConfig.siteName` / `siteConfig.seoKeywords` を編集してください。
「〇〇市」「〇〇県」という表記は各データファイル・各ページの文言に埋め込まれているため、
実際の地域名が決まった際は、`data/` 配下のファイルと `app/` 配下の `metaDescription` 等を
一括置換（お使いのエディタの「フォルダ内を検索して置換」機能など）していただくのが確実です。

### サービス内容（住宅リフォーム、新築住宅、法人向け工事、不動産サービスなど）
[`data/services.ts`](data/services.ts) を編集してください。1サービス＝1オブジェクトになっており、
概要・お悩み例・対応例・強み・FAQ・関連する施工事例カテゴリーなどをここでまとめて管理しています。
文言を変えるだけで、トップページのカード・サービス一覧ページ・詳細ページすべてに反映されます。

### 施工事例
[`data/projects.ts`](data/projects.ts) の配列に追加・編集してください。1件追加すると、
トップページの新着表示・一覧ページ・詳細ページ（`/projects/[slug]`）が自動的に生成されます。
`slug` はURLの一部になるため、半角英数字とハイフンのみを使用してください。

### 不動産物件情報
[`data/properties.ts`](data/properties.ts) の配列に追加・編集してください。`dealType` を
`"sale"`（売買）または `"rent"`（賃貸）に設定すると、一覧ページの売買/賃貸タブや絞り込みに
自動的に反映されます。

### よくある質問
[`data/faq.ts`](data/faq.ts) の配列に追加し、`data/services.ts` の各サービスの `faqIds` に
IDを追加すると、該当サービスページのFAQに表示されます。

### お知らせ
[`data/news.ts`](data/news.ts) の配列の先頭に新しい項目を追加してください。

### 画像
[`data/placeholderImages.ts`](data/placeholderImages.ts) に現在は仮画像（Unsplash）のURLを
まとめています。正式な写真が用意でき次第、このファイルの値を差し替えるか、`public/images/` に
画像ファイルを配置して、そのパス（例: `/images/hero-top.jpg`）に書き換えてください。
画像を差し替える箇所はこのファイル1つに集約されているため、迷ったらまずここを確認してください。

### OGP画像
`public/images/og-default.jpg`（1200×630px推奨）は現状ファイルが未配置です。SNSでシェアされた際の
サムネイル表示に使われるため、実際の画像が用意でき次第このパスに配置してください。

### お問い合わせフォームの送信先
現時点では送信先（バックエンド）が未確定のため、[`lib/submitContactForm.ts`](lib/submitContactForm.ts)
はフォーム内容をコンソールに出力するだけのダミー実装になっています。入力チェック・確認画面・完了画面は
実装済みなので、送信先が決まったら、この関数の中身を実際のAPI呼び出し（例: `fetch("/api/contact", ...)`
や外部フォームサービスへの送信）に差し替えるだけで接続できます。

### 会社概要ページの未確定項目
[`app/company/page.tsx`](app/company/page.tsx) と `data/siteConfig.ts` 内に
「（会社名 未確定）」「（住所 未確定）」のように、確定後に置き換える項目であることが
分かる形で仮表示しています。検索して該当箇所を確認・置換してください。
